## Implementation Phases

### Phase A вЂ” Backend filter params

Extend three endpoints to accept optional `tag` and `payment_method` query params.

**Files**: `backend/src/handlers/pwa.ts`

Rules:
- `payment_method` в†’ add `.eq('payment_method', paymentMethod)` to the Supabase query (efficient column filter)
- `tag` в†’ fetch all txs for the date range, then filter in-Worker: a transaction matches if `tx.tags` OR any of `tx.transaction_items[].tags` contains the plain tag (i.e., tag value does not contain `:`)
- Both filters compose with AND logic (apply both when both are present)
- No change to response shape

**Tests** (`backend/tests/handlers/pwa-filters.test.ts`):
- `tag` filter: tx with matching tx-level tag included; tx with matching item-level tag included; tx with no matching tag excluded
- `payment_method` filter: tx with matching method included; others excluded
- Both filters together: only tx matching both included

### Phase B вЂ” Frontend time model

Replace `WindowOption` with `TimeBase + offset` in `useSummary.ts`.

**Files**: `pwa/src/hooks/useSummary.ts`

Changes:
- Add `export type TimeBase = 'week' | 'month' | 'year' | 'all'`
- Add `export function timeBaseToRange(base: TimeBase, offset: number): { from: string; to: string; label: string }`
  - week: SunвЂ“Sat calendar week; label `"D MMM"` or `"DвЂ“D MMM"` format
  - month: 1stвЂ“last day; label `"MMM YYYY"` (Chinese month names: `['1жњ€','2жњ€',...,'12жњ€']`)
  - year: Jan 1вЂ“Dec 31; label `"YYYY"`
  - all: `2000-01-01` to today; label `"е…ЁйѓЁ"`
- Update `useSummaryData(base, offset, tag?, paymentMethod?)` вЂ” queryKey includes all four params
- Update `useSubcategoryData(major, base, offset, tag?, paymentMethod?)`
- Update `useTransactions(base, offset, category?, tag?, paymentMethod?)` вЂ” remove `WindowOption`-based `WINDOW_LIMIT` map; use a fixed limit of 500 (week) / 300 (month) / 2000 (year) based on `base`
- Keep `useTransactionPeriods` and `useMonthTransactions` unchanged (used by 'all' mode)
- Remove `WindowOption` type and `windowToDates()` вЂ” no longer needed

### Phase D вЂ” PeriodPicker component

**File**: `pwa/src/components/PeriodPicker.tsx` (new)

Internal state: `step: 1 | 2`, `selectedYear: number`

- Current year scrolled into view on open
- Tap year в†’ `year` mode: compute `yearOffset = selectedYear - currentYear`, call `onSelect(yearOffset)` and `onClose()`; else set `selectedYear` and advance to step 2

Month grid (step 2, month mode):
- 4Г—3 grid (3 cols Г— 4 rows): JanвЂ“Dec
- Future months in `currentYear` disabled (month index > `new Date().getMonth()`)
- Tap в†’ compute offset = `(selectedYear - y) * 12 + (monthIdx - m)`, call `onSelect` + `onClose()`

Week list (step 2, week mode):
- Month tab strip (JanвЂ“Dec) вЂ” default to current month in selected year
- List of week rows within the selected month (each row: `"D MMM вЂ“ D MMM"`)
- Future weeks disabled
- Tap в†’ compute week offset from current week, call `onSelect` + `onClose()`

Modal chrome: fixed `inset-0 bg-black/40 z-50`, inner panel slides up from bottom (or centered sheet). `вњ•` button and backdrop tap close via `onClose`.
