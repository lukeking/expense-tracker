# Implementation Plan: E-Invoice CSV Import + /amend

**Branch**: `004-einvoice-csv-import` | **Date**: 2026-05-09 | **Spec**: [spec.md](spec.md)

## Summary

Two features shipped sequentially. **Phase 1**: `/amend` Discord command — corrects an existing transaction's NTD amount via candidate-list UX (same pattern as `/fee`/`/refund`), primarily for foreign-currency forex settlement corrections. **Phase 2**: `/import` Discord command — accepts a government e-invoice CSV attachment, parses it (UTF-8/Big5, ROC calendar dates), matches invoices to existing transactions (exact amount ±2 days, with a secondary ±5% "likely forex" pass), auto-creates records for unmatched invoices, and runs a post-import reconciliation pass over all previously-held forex invoices.

---

## Technical Context

**Language/Version**: TypeScript (Node-compatible ESM, CF Workers runtime)
**Primary Dependencies**: Hono (router), Supabase JS client, Gemini API, `big5` npm package (CSV encoding fallback)
**Storage**: Supabase/PostgreSQL — two new tables (`import_runs`, `invoices`), three new columns on `transactions`
**Testing**: Vitest + `@cloudflare/vitest-pool-workers` (Miniflare)
**Target Platform**: Cloudflare Workers (Unbound plan — 30 s CPU wall time)
**Performance Goals**: SC-001 — 20-invoice CSV processed + summary visible within 30 seconds
**Constraints**: CF Workers 128 MB memory; all heavy work in `ctx.waitUntil()`; Discord interaction must respond within 3 seconds (type:5 deferred)
**Scale/Scope**: Single user; expected ~20–100 invoices per import; up to 500 invoices for large runs

---

## Constitution Check

- [x] **I. Simplicity-First** — No new abstractions beyond what's needed. `/amend` reuses `findParentCandidates`. Import uses a straightforward pipeline. `big5` package added for Big5 decode only; no other new dependencies.
- [x] **II. Offline-First on Android** — Android not in scope for this feature. `/amend` Discord-only per user decision. No Android changes.
- [x] **III. Serverless Boundary Compliance** — Both `/amend` and `/import` return `type:5` immediately. All CSV fetch, parse, DB writes, and Gemini calls run inside `ctx.waitUntil()`. Large CSV batch processing keeps CPU well within 30 s wall time.
- [x] **IV. Automation Over Manual Input** — Core purpose: auto-match invoices, auto-create missed transactions, auto-reconcile held forex invoices. User interaction reduced to uploading the CSV once every 2 months.
- [x] **V. Security at System Boundaries** — No new secrets needed. Discord ed25519 verification already in place. Supabase service role key stays server-side only.

## Implementation Phases

### Phase A — `/amend` Discord command (implement first)

**Goal**: End-to-end `/amend` working in Discord. No Android. No CSV work.

Tasks:
1. Add `amendTransactionAmount(supabase, txId, newAmount)` to `queries.ts`
2. Register `/amend` command in `register-commands.ts`
3. Add `/amend` handler in `discord.ts` — type:5 deferred, `findParentCandidates`, patch with buttons
4. Add `amend_select` component handler — type:7 UPDATE_MESSAGE, call `amendTransactionAmount`, show confirmation + budget summary
5. Add `amend_retype` component handler — type:9 MODAL
6. Add `amend_modal` submit handler — type:6 DEFERRED_UPDATE_MESSAGE, re-search, patch
7. Add `amend_cancel` component handler — type:7 UPDATE_MESSAGE, clear buttons
8. Tests: amendTransactionAmount, /amend flow (type:5 response, custom_id encoding, UPDATE_MESSAGE confirmation)

### Phase B — Schema migration

**Goal**: New tables and columns in Supabase.

Tasks:
9. Write `backend/supabase/migrations/004_einvoice_import.sql`
10. Apply migration to Supabase (manual step; document in quickstart)
11. Add `Invoice`, `ImportRun`, `RawInvoiceRow`, `ParsedInvoice`, `InvoiceMatchStatus` types to `types.ts`

### Phase C — CSV parser service

**Goal**: Stateless CSV parsing with encoding detection, ROC date conversion, row grouping.

Tasks:
12. Write `src/services/csv-parser.ts`:
    - `decodeCSVBuffer(buffer: ArrayBuffer): string` (UTF-8 strict → Big5 fallback)
    - `parseROCDate(raw: string): Date` (adds 1911)
    - `parseCSVRows(csv: string): RawInvoiceRow[]` (header validation, row parsing)
    - `groupInvoices(rows: RawInvoiceRow[]): ParsedInvoice[]` (group by 発票號碼, aggregate items)
13. Install `big5` npm package
14. Tests: encoding fallback, ROC date conversion, voided/zero filtering, multi-item grouping, malformed row tolerance

### Phase D — Invoice matcher service

**Goal**: DB-backed matching pipeline.

Tasks:
15. Add DB query functions to `queries.ts`:
    - `createImportRun(supabase, fileName): ImportRun`
    - `updateImportRun(supabase, runId, counters)`
    - `findExistingInvoiceNumbers(supabase, invoiceNumbers: string[]): string[]`
    - `findMatchingExpenseTransaction(supabase, netAmount, invoiceDate): Transaction | null` (exact, ±2 days)
    - `findForexCandidateTransaction(supabase, netAmount, invoiceDate): Transaction | null` (±5%, ±2 days)
    - `insertInvoice(supabase, invoice, importRunId, matchStatus, matchedTxId?): Invoice`
    - `enrichTransaction(supabase, txId, invoiceData)` (UPDATE is_matched, invoice_number, seller_name, seller_tax_id, matched_invoice_id)
    - `findAllHeldForexInvoices(supabase): Invoice[]`
    - `resolveHeldInvoice(supabase, invoiceId, txId, matchStatus)`
16. Write `src/services/invoice-matcher.ts`:
    - `runImportPipeline(supabase, invoices, fileName, gemini): ImportRun` — orchestrates steps A–J from contracts doc
    - `runReconciliationPass(supabase, gemini): number` — re-evaluates all held_forex invoices, returns resolved count
17. Tests: primary match, secondary forex match, dedup skip, voided skip, zero skip, auto-create, reconciliation pass resolves held after amend

### Phase E — Discord `/import` command

**Goal**: End-to-end `/import` working in Discord.

Tasks:
18. Register `/import` command in `register-commands.ts` (type 11 attachment option)
19. Extend `DiscordInteraction` type with `resolved.attachments`
20. Add `/import` handler in `discord.ts`:
    - Return type:5 immediately
    - `ctx.waitUntil`: fetch CSV bytes → parse → run pipeline → patch summary
21. Write `formatImportSummary(run: ImportRun, unmatchedTxs: Transaction[]): string`
22. Tests: /import routing, attachment URL extraction, error paths (bad format, 0 rows, fetch failure)

### Phase F — Periodic reminder (FR-010)

**Goal**: Cron Trigger sends Discord reminder every 2 months.

Tasks:
23. Add cron schedule to `wrangler.toml`: `0 9 1 */2 *` (9am on the 1st of every other month)
24. Add cron handler in `index.ts` that sends Discord reminder message to `DISCORD_CHANNEL_ID`
25. Reminder message includes date of last import run (query `import_runs` ORDER BY created_at DESC LIMIT 1)

## Complexity Tracking

| Addition | Why Needed | Simpler Alternative Rejected Because |
|----------|------------|--------------------------------------|
| `big5` npm package | Government CSV may use Big5 encoding; CF Workers TextDecoder doesn't support it | Node.js `iconv-lite` uses Node APIs unavailable in CF Workers |
| 2 new DB tables | `import_runs` for audit; `invoices` for dedup, held state, reconciliation | Storing invoice state on transactions only would make dedup by invoice_number impossible across runs |
