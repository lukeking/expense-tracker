# Implementation Plan: Explicit Payment Method Option for /expense

**Branch**: `006-expense-payment-option` | **Date**: 2026-05-10 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/006-expense-payment-option/spec.md`

## Summary

Add a `payment_method` Discord option (5 predefined choices) to the `/expense` slash command, replacing keyword-based payment extraction from the description text. The Discord handler reads the option value directly and displays the Chinese label inline in the confirmation message. No database migration is required — the `payment_method` column with CHECK constraint already exists.

**Scale/Scope**: Single-user personal tool

## Implementation Design

### 1. `register-commands.ts` — Add `payment_method` option to `/expense`

Change the command `description` from "記錄一筆現金支出" → "記錄一筆支出".

Add a new option after `description`:

```typescript
{
  name: 'payment_method',
  description: '付款方式（預設：現金）',
  type: 3, // STRING
  required: false,
  choices: [
    { name: '現金',   value: 'cash' },
    { name: '信用卡', value: 'credit_card' },
    { name: '悠遊卡', value: 'easy_card' },
    { name: '銀行轉帳', value: 'bank_account' },
    { name: '行動支付', value: 'prepaid_wallet' },
  ],
}
```

### 2. `discord.ts` — Read `payment_method` option and update confirmation

In `handleExpenseCommand`:

a. **Read the option** before the async block:
   ```typescript
   const paymentMethod = (
     (options.find((o) => o.name === 'payment_method')?.value as string) ?? 'cash'
   ) as PaymentMethod;
   ```

b. **Pass to `insertTransaction`** instead of `parsed.payment_method`:
   ```typescript
   payment_method: paymentMethod,
   ```

c. **Add display label map** (module-level const):
   ```typescript
   const PAYMENT_METHOD_LABEL: Record<PaymentMethod, string> = {
     cash: '現金',
     credit_card: '信用卡',
     easy_card: '悠遊卡',
     bank_account: '銀行轉帳',
     prepaid_wallet: '行動支付',
   };
   ```

d. **Update confirmation message** to append label inline on the amount line:
   ```
   💰 金額：$${amount} [${PAYMENT_METHOD_LABEL[paymentMethod]}]
   ```

### 3. `gemini.ts` — Remove payment keyword parsing from Discord prompt

The `COMMON_PROMPT_RULES` contains payment_method mapping used by both `SYSTEM_PROMPT` (Discord) and `RAW_TEXT_SYSTEM_PROMPT` (Android). Android must remain unchanged.

- Rename `COMMON_PROMPT_RULES` → `ANDROID_PROMPT_RULES` (used only by `RAW_TEXT_SYSTEM_PROMPT`)
- Create `DISCORD_PROMPT_RULES` — identical to the old `COMMON_PROMPT_RULES` but with the payment_method mapping block removed
- Update `SYSTEM_PROMPT` to use `DISCORD_PROMPT_RULES`
- `RAW_TEXT_SYSTEM_PROMPT` continues to use `ANDROID_PROMPT_RULES`
- `GeminiParseResult` type and function signatures are unchanged; `parseExpenseText` still returns `payment_method` (will be `'cash'` when no keyword in description — acceptable since the Discord handler now ignores it)

### 4. Tests

**`discord.test.ts`** — update `/expense` test:
- Add `payment_method: 'credit_card'` to the options fixture and assert it is passed to `insertTransaction`
- Add test: omitting `payment_method` defaults to `'cash'`

**`gemini.test.ts`** — update `parseExpenseText` tests:
- Verify that passing a description containing "信用卡" no longer returns `payment_method: 'credit_card'` (returns `'cash'` instead, since keyword mapping is removed from `DISCORD_PROMPT_RULES`)
- Keep all `parseRawExpenseText` payment mapping tests unchanged
